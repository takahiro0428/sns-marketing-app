# ブログ運用ガイド

## シリーズ概要

AIネイティブ開発方法論の進化過程・思考・改善を発信する継続シリーズ。
完結を定義せず、時系列で進化を記録し続ける。

## 対象媒体

### Note: 「AIチーム開発記 — 経営者がAI8人とチームを組んだ話」

- **ペルソナ:** テック経営者・開発組織リーダー(CTO, VPoE, テックリード, PM, スタートアップ経営者)
- **トーン:** 実践者の独白。「私はこう考えた」「こうやってみた」
- **構成パターン:** きっかけ → 仮説 → やってみた結果 → 学び → 次への問い
- **焦点:** 経営者の組織設計経験がAI運用に転用される思考過程

### Zenn: 「AIネイティブ開発実践ガイド」

- **ペルソナ:** AI活用を実践する中上級エンジニア(3年以上, Claude Code / Copilot利用者)
- **トーン:** 技術記事。再現可能性を重視
- **構成パターン:** 課題提起 → アプローチ → 実装/設定 → 検証結果 → 応用ポイント
- **焦点:** 具体的な実装パターン、設定例、再現手順

## コンテンツ差別化

| 観点 | Note | Zenn |
|------|------|------|
| 同じ題材の場合 | 「なぜそう考えたか」経営者の思考過程 | 「どうやるか」エンジニアの実装手順 |
| 失敗の扱い | 組織運営の文脈でストーリーとして語る | 技術的原因分析として記述 |
| 図表 | 概念図・関係図(シンプル) | Mermaid図・コード例・設定ファイル |
| 読後感 | 「自分のチーム/組織でも考えてみよう」 | 「手元で試してみよう」 |
| 相互リンク | Zenn記事を「詳細はこちら」で参照 | Note記事を「背景はこちら」で参照 |

## ファイル命名規則

```
blog/
├── note/
│   └── {NNN}-{slug}.md    # 例: 001-ai-team-development.md
└── zenn/
    └── {NNN}-{slug}.md    # 例: 001-8-role-architecture.md
```

## 招集ロール

| ロール | 担当領域 |
|--------|----------|
| ナビゲーター | 全体構成・記事計画の壁打ち・方向性判定 |
| テクニカルライター | 記事本文の執筆・構造化・読者向け表現 |
| エデュケーター | 方法論の正確性検証・読者への教育的価値の担保 |
| ユーザー・運用サポート | ペルソナ検証・読者視点でのレビュー |
